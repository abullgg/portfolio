> project: PhishGuard

A full-stack machine learning application designed to detect zero-day phishing URLs through real-time lexical analysis.

Instead of relying on static blocklists, the system uses a TF-IDF vectorizer with a Logistic Regression model to evaluate URL patterns and assign confidence-based predictions. The model is loaded globally to eliminate runtime latency and optimize inference speed.

The platform includes a RESTful API for integration and an internal dashboard for tracking predictions, enabling monitoring of threat patterns and system performance.

Focus: real-time ML inference, backend architecture, and security-focused application design.

Tech: Python, Django, Scikit-learn, Pandas, Tailwind CSS
> project: cmnty-backend

A robust RESTful API designed to power scalable community event management, club hosting, and user registrations.

Instead of basic CRUD, the system employs strict role-based access control and transactional database locks to enforce real-time event capacity limits, eliminating vulnerabilities like overlapping concurrent registrations. Security is handled natively via stateless JSON Web Tokens (JWT) and BCrypt password hashing.

The platform includes a complex data delivery mechanism utilizing custom DTOs, dynamic JPA pagination, and precise ISO date filtering to optimize payload size and performance for downstream frontend clients.

Focus: Enterprise REST API design, secure application architecture, and transactional data integrity.

Tech: Java 17, Spring Boot, PostgreSQL, Spring Security, Hibernate/JPA, Mockito
> project: rag-document-assistant

A full-stack document intelligence tool that lets you upload a PDF or TXT file and ask natural language questions against it — all processed locally on your machine.

Instead of cloud-dependent APIs, the system runs local LLM inference via Ollama with Qwen3 4B (featuring a massive 32,768-token context window). Qwen3 optimizations include suppressing internal chain-of-thought token generation (think=False and think-tag post-processing strip) to minimize latency, and utilizing structured XML context tags to enhance grounded attention accuracy.

The platform features hybrid semantic-keyword retrieval combining FAISS vector search with BM25 ranking, re-ranked results isolated per upload, and background job tracking for large file processing. By exploiting the larger context window, the retrieval budget has been raised to a generous 12,000-character ceiling, which is populated by a wider cross-encoder pool (15 candidates re-ranked down to 8) and fed by larger, less fragmented chunk presets.

Frontend delivers a ChatGPT-style interface with full conversation history, markdown rendering, live health monitoring, and Railway-inspired dark design.

Focus: Cost-efficient local inference, hybrid retrieval architecture, context budget optimization, and real-time document Q&A.

Tech: FastAPI, pdfplumber, FAISS, BM25, Ollama (Qwen3 4B), Next.js, TypeScript, Vanilla CSS
